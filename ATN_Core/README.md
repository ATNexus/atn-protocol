# ATN Core - AgentTrust Nexus 创世内核

## 项目概述

ATN Core 是 AgentTrust Nexus 协议的核心实现，提供了一套完整的去中心化Agent交互基础设施，包括身份管理、资金托管和争议仲裁。

## 架构设计

```
ATN_Core/
├── escrow/              # 资金托管模块
│   └── atomic_escrow.py # 原子托管合约
├── reputation/          # 声誉系统模块
│   └── agent_identity.py # Agent身份和SBT管理
├── arbitration/         # 仲裁模块
│   └── schelling_jury.py # 谢林点仲裁逻辑
├── common/              # 公共工具
└── main.py             # 集成演示
```

## 核心机制详解

### 1. Agent身份系统 (AgentIdentity)

- **ECDSA ID生成**: 基于公钥生成唯一标识符
- **SBT信用档案**: 不可转让的灵魂绑定代币，记录Agent历史表现
- **信用评分**: 初始100分，成功任务+5，失败任务-10

### 2. 原子托管合约 (AtomicEscrow)

- **资金隔离**: 任务资金进入独立隔离池，与Agent余额分离
- **超时保护**: 15分钟超时机制，自动退款保障买方权益
- **签名验证**: 任务完成需要有效的加密签名才能释放资金

### 3. 谢林点仲裁 (JuryVote)

- **随机选裁**: 从注册裁判中随机选择3名
- **质押机制**: 每名裁判需质押0.1 SOL参与
- **多数获胜**: 2/3多数派决定结果
- **奖惩分明**: 少数派失去质押，多数派获得奖励

## Solana链上部署所需接口

### 1. 智能合约程序 (Programs)

| 程序名称 | 功能描述 | 所需接口 |
|---------|---------|---------|
| `atn_identity` | Agent身份管理 | `create_agent`, `update_sbt`, `get_profile` |
| `atn_escrow` | 资金托管 | `initialize_escrow`, `lock_funds`, `release_funds`, `refund` |
| `atn_jury` | 争议仲裁 | `register_juror`, `initiate_dispute`, `cast_vote`, `resolve` |

### 2. 账户结构 (Accounts)

```rust
// Agent账户
pub struct AgentAccount {
    pub agent_id: String,
    pub public_key: Pubkey,
    pub sbt_score: u32,
    pub total_tasks: u64,
    pub successful_tasks: u64,
    pub bump: u8,
}

// 托管账户
pub struct EscrowAccount {
    pub task_id: String,
    pub buyer: Pubkey,
    pub seller: Pubkey,
    pub amount: u64,
    pub status: EscrowStatus,
    pub created_at: i64,
    pub timeout_at: i64,
    pub bump: u8,
}

// 争议账户
pub struct DisputeAccount {
    pub dispute_id: String,
    pub task_id: String,
    pub complainant: Pubkey,
    pub respondent: Pubkey,
    pub jurors: [Pubkey; 3],
    pub votes: Vec<Vote>,
    pub status: DisputeStatus,
    pub bump: u8,
}
```

### 3. 需要您配合提供的接口

#### A. Solana程序部署

1. **程序ID (Program IDs)**
   - `atn_identity` 程序的部署地址
   - `atn_escrow` 程序的部署地址
   - `atn_jury` 程序的部署地址

2. **代币合约**
   - 协议代币的Mint地址（如果使用自定义代币）
   - 或确认使用原生SOL

#### B. RPC节点和索引服务

1. **RPC端点**
   - 主网/测试网RPC URL
   - 支持WebSocket订阅（用于实时事件监听）

2. **索引服务** (可选但推荐)
   - The Graph子图或类似服务，用于高效查询历史数据

#### C. 前端/客户端SDK

1. **钱包适配器**
   - Phantom, Solflare等钱包的集成配置

2. **交易签名服务**
   - 用于Agent自动签名的密钥管理服务

#### D. 外部数据源接口

1. **价格预言机** (用于仲裁时验证市场价格)
   - Pyth Network或Chainlink的SOL/USD价格源

2. **API服务** (用于特定任务类型)
   - 航班数据API (如需要验证机票价格)
   - 其他第三方数据源

### 4. 安全考虑

- **多签控制**: 协议升级和紧急暂停需要多签
- **审计**: 智能合约需要专业安全审计
- **漏洞赏金**: 建立漏洞赏金计划
